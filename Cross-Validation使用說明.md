# Cross-Validation 使用說明

## 🎯 什麼是 Cross-Validation？

Cross-Validation（交叉驗證）是一種更可靠的模型評估方法，特別適合資料量有限的情況。

### 傳統 80/20 分割 vs Cross-Validation

**傳統方法（80/20）：**
```
全部資料（109 筆）
├── 訓練集：87 筆（80%）→ 訓練模型
└── 測試集：22 筆（20%）→ 評估模型

問題：
- 20% 的資料永遠不參與訓練（浪費）
- 只評估一次（可能運氣好/壞）
- 不知道模型的穩定性
```

**Cross-Validation（5-Fold）：**
```
全部資料（109 筆）分成 5 份

第1折：訓練 87 筆，驗證 22 筆 → 評估1
第2折：訓練 87 筆，驗證 22 筆 → 評估2
第3折：訓練 87 筆，驗證 22 筆 → 評估3
第4折：訓練 87 筆，驗證 22 筆 → 評估4
第5折：訓練 87 筆，驗證 22 筆 → 評估5

最後：平均 5 次評估結果 → 更可靠的評估

優勢：
✓ 每筆資料都被用來訓練和測試
✓ 評估 5 次，更可靠
✓ 知道模型的穩定性（標準差）
✓ 最終用全部 109 筆訓練模型
```

## 📊 輸出結果

### 1. 控制台輸出

**Cross-Validation 結果：**
```
✓ Cross-Validation 結果:
  訓練集 MAE:  3.2456 ± 0.1234
  驗證集 MAE:  5.6789 ± 0.8765
  訓練集 RMSE: 4.5678 ± 0.2345
  驗證集 RMSE: 7.8901 ± 1.2345
  訓練集 R²:   0.8765 ± 0.0234
  驗證集 R²:   0.6543 ± 0.1234
```

**解讀：**
- **平均值**：模型的預期表現
- **標準差（±）**：模型的穩定性
  - 標準差小（< 2）→ 模型穩定
  - 標準差大（> 5）→ 模型不穩定

### 2. 訓練結果圖表

**CV 模式的圖表包含：**

1. **特徵重要性**（左上）
   - 顯示最重要的 10 個特徵

2. **CV 分數分布**（右上）
   - 顯示每一折的 MAE
   - 可以看出模型在不同資料子集上的表現

3. **R² 分數分布**（左下）
   - 顯示每一折的 R²
   - 評估模型的解釋能力

4. **模型穩定性**（右下）
   - 顯示各指標的標準差
   - 標準差越小，模型越穩定

## 🎯 何時使用 CV？

| 資料量 | 建議方法 | 原因 |
|--------|----------|------|
| < 100 筆 | **5-10 Fold CV** | 資料太少，必須充分利用 |
| 100-200 筆 | **5 Fold CV** ✅ | 最佳選擇 |
| 200-500 筆 | **5 Fold CV** 或 80/20 | CV 仍然更好 |
| > 500 筆 | 80/20 或 CV | 都可以 |

**你的情況（109 筆）：強烈建議使用 5-Fold CV**

## 📈 CV 結果解讀

### 範例結果

```
驗證集 MAE:  5.67 ± 0.89
驗證集 RMSE: 7.89 ± 1.23
驗證集 R²:   0.65 ± 0.12
```

### 解讀

**1. 平均 MAE = 5.67**
- 預測誤差平均約 5.67%
- 這是模型的預期表現

**2. 標準差 = 0.89**
- 不同折之間的 MAE 差異約 0.89%
- 標準差小 → 模型穩定
- 標準差大 → 模型不穩定（可能過擬合）

**3. R² = 0.65 ± 0.12**
- 模型可以解釋 65% 的變異
- 標準差 0.12 表示在不同資料子集上表現一致

### 判斷標準

**模型穩定性：**
- MAE 標準差 < 2 → ✅ 非常穩定
- MAE 標準差 2-5 → ⚠️ 尚可
- MAE 標準差 > 5 → ❌ 不穩定

**模型表現：**
- 驗證集 MAE < 5% → ✅ 優秀
- 驗證集 MAE 5-10% → ⚠️ 尚可
- 驗證集 MAE > 10% → ❌ 需要改進

**過擬合檢測：**
- 訓練集 MAE - 驗證集 MAE < 2% → ✅ 無過擬合
- 訓練集 MAE - 驗證集 MAE 2-5% → ⚠️ 輕微過擬合
- 訓練集 MAE - 驗證集 MAE > 5% → ❌ 嚴重過擬合

## 🔧 常見問題

### Q1: CV 訓練時間會更長嗎？

**A:** 是的，約 5 倍時間（5-Fold）。

- 傳統訓練：1 分鐘
- 5-Fold CV：5 分鐘

但這是值得的，因為評估更可靠。

### Q2: 最終模型是哪一個？

**A:** 用全部資料訓練的模型。

CV 流程：
1. 用 CV 評估模型表現（5 次）
2. 用全部 109 筆資料訓練最終模型
3. 用 CV 的結果估計最終模型的表現

### Q3: CV 和 80/20 哪個更好？

**A:** 幾乎總是 CV 更好。

除非：
- 資料量非常大（> 10,000 筆）
- 訓練時間非常長
- 只需要快速估計

### Q4: 應該用幾折？

**A:** 通常 5 折就夠了。

- 5 Fold：平衡速度和可靠性（推薦）
- 10 Fold：更可靠，但更慢
- 3 Fold：更快，但不太可靠

### Q5: CV 結果和實際預測會一樣嗎？

**A:** 不會完全一樣，但很接近。

CV 結果是模型的**預期表現**，實際預測時：
- 可能稍好（運氣好）
- 可能稍差（運氣差）
- 但平均來說會接近 CV 結果

## 💡 最佳實踐

### 1. 定期重新訓練

每增加 20-30 筆資料，重新訓練一次：

```bash
python train_qt_xgboost.py --cv
```

### 2. 記錄 CV 結果

建立一個表格記錄每次訓練的 CV 結果：

| 日期 | 資料量 | 驗證集 MAE | 驗證集 R² | 標準差 |
|------|--------|-----------|----------|--------|
| 2026-02-17 | 109 | 5.67 | 0.65 | 0.89 |
| 2026-03-01 | 130 | 5.12 | 0.72 | 0.67 |
| 2026-03-15 | 150 | 4.89 | 0.78 | 0.54 |

### 3. 觀察趨勢

隨著資料量增加，應該看到：
- ✅ MAE 下降（更準確）
- ✅ R² 上升（解釋力更強）
- ✅ 標準差下降（更穩定）

### 4. 結合可信度

當資料量達到 150+ 筆時：
- CV 結果會更可靠
- 可信度計算會更有意義
- 可以開始信賴高可信度的預測
