# 訓練流程說明

## 新的訓練流程（方式 1：80/20 評估 + 全資料訓練）

### 問題背景

**舊版本的問題：**
- 載入 125 筆資料
- 分割成 80/20 → 訓練集 100 筆，測試集 25 筆
- 用 100 筆訓練模型
- 用 25 筆測試模型
- **儲存這個只用 100 筆訓練的模型** ❌

**問題：**
- 浪費了 25 筆資料（20%）
- 最終模型沒有從這 25 筆資料中學習
- 模型的表現比它應有的表現更差

### 新版本的解決方案

**訓練流程：**

```
步驟 1-2: 載入並預處理資料（125 筆）

步驟 3: 分割資料
  ├─ 訓練集：100 筆（80%）
  └─ 測試集：25 筆（20%）

步驟 4: 用 100 筆訓練模型
  └─ 目的：建立一個模型來評估

步驟 5: 用 25 筆測試模型
  ├─ 測試集 MAE: 2.48%
  ├─ 測試集 RMSE: 3.92%
  └─ 測試集 R²: 0.74
  └─ 目的：評估模型的泛化能力

步驟 6: 視覺化結果
  └─ 生成訓練結果圖表

步驟 6.5: 用全部 125 筆重新訓練最終模型 ✓
  ├─ 使用相同的模型參數
  ├─ 使用全部資料訓練
  └─ 這是最終要儲存的模型

步驟 7: 儲存最終模型
  └─ 儲存用 125 筆訓練的模型
```

### 關鍵優勢

1. **充分利用資料**
   - 最終模型使用全部 125 筆資料
   - 沒有浪費任何資料

2. **可靠的評估**
   - 步驟 5 的測試集結果告訴我們模型的預期表現
   - 這個評估是可靠的（因為測試集沒有參與訓練）

3. **更好的模型**
   - 用 125 筆訓練比用 100 筆訓練更好
   - 資料量增加 25%，模型表現會提升

### 實際範例

**步驟 5 的評估結果（用 100 筆訓練，25 筆測試）：**
```
測試集評估:
  - RMSE: 3.9204
  - MAE: 2.4757
  - R² Score: 0.7404
```

**步驟 6.5 的最終模型（用 125 筆訓練）：**
```
使用全部資料: 125 筆
  （之前評估時只用了 100 筆訓練）

最終模型在全部資料上的表現:
  - RMSE: 0.2316
  - MAE: 0.1587
  - R² Score: 0.9995

註: 這是訓練集表現，實際泛化表現請參考步驟 5 的測試集結果
```

### 如何解讀結果

**步驟 5 的測試集結果（MAE: 2.48%）：**
- 這是模型的**預期表現**
- 當你用這個模型預測新股票時，預期誤差約 2.48%
- 這個評估是可靠的（測試集沒有參與訓練）

**步驟 6.5 的訓練集結果（MAE: 0.16%）：**
- 這只是模型在訓練資料上的表現
- 不代表實際預測能力（因為模型已經「看過」這些資料）
- 僅供參考，不要過度解讀

**最終模型的預期表現：**
- 約 2.48%（來自步驟 5）
- 可能稍好一些（因為用了更多資料訓練）
- 但不會好到 0.16%（那是過擬合的表現）

### 使用方法

**訓練模型：**
```bash
# 使用批次檔
快速訓練.bat

# 或使用命令列
python train_qt_xgboost.py
```

**訓練過程會自動：**
1. 用 80/20 分割評估模型
2. 用全部資料重新訓練
3. 儲存用全部資料訓練的模型

**不需要任何額外操作！**

### 與 Cross-Validation 的比較

| 特性 | 方式 1（80/20 + 全資料） | Cross-Validation |
|------|------------------------|------------------|
| 評估可靠性 | 中等（只評估 1 次） | 高（評估 5 次） |
| 訓練時間 | 快（訓練 2 次） | 慢（訓練 6 次） |
| 最終模型 | 用全部資料訓練 | 用全部資料訓練 |
| 穩定性評估 | 無 | 有（標準差） |
| 實作複雜度 | 簡單 | 中等 |
| 適用情況 | 資料量 > 200 筆 | 資料量 < 200 筆 |

**你的情況（125 筆）：**
- 方式 1 已經足夠好
- 如果想要更可靠的評估，可以考慮 CV
- 但方式 1 已經解決了「浪費資料」的問題

### 總結

**修改前：**
- ❌ 只用 100 筆訓練最終模型
- ❌ 浪費 25 筆資料

**修改後：**
- ✅ 用 125 筆訓練最終模型
- ✅ 充分利用所有資料
- ✅ 仍然有可靠的評估（來自測試集）
- ✅ 模型表現更好（因為用了更多資料）

這是一個簡單但有效的改進！
