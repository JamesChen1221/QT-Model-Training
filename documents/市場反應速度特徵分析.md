# 市場反應速度特徵分析

## 問題描述

### 核心問題：市場反應速度的差異

**情境說明：**

```
訓練資料 A：
- 特徵：財報超預期 +10%、技術指標強勢、產業熱門
- 0.5小時最高價：+15%
- 實際：開盤後 10 分鐘就到達 +15%

待預測資料 B：
- 特徵：財報超預期 +10%、技術指標強勢、產業熱門（與 A 相似）
- 預測 0.5小時最高價：+15%
- 實際：3 小時後才到達 +15%，0.5小時最高價只有 +8%
```

**問題根源：**
- 兩筆資料的特徵相似
- 最終都到達相同的價格高度
- 但市場反應速度不同
- 導致預測的時間點（0.5小時）內，實際價格與預測不符

**影響：**
- 對當沖交易來說，反應速度至關重要
- 快速反應 = 可以快速進出，風險低
- 緩慢反應 = 需要等待更久，風險高

## 缺少的關鍵特徵

### 1. 流動性相關特徵 ⭐⭐⭐（最重要）

**為什麼重要：**
- 流動性高 → 市場反應快（買賣容易，價格快速調整）
- 流動性低 → 市場反應慢（買賣困難，價格緩慢調整）

**建議特徵：**

| 特徵名稱 | 說明 | 計算方式 | 重要性 |
|---------|------|---------|--------|
| 平均日交易量 | 過去 20 天平均交易量 | `mean(volume[-20:])` | ⭐⭐⭐ |
| 平均日交易金額 | 過去 20 天平均交易金額 | `mean(volume * price)` | ⭐⭐⭐ |
| 相對交易量 | 當日 vs 平均 | `today_volume / avg_volume` | ⭐⭐⭐ |
| 盤前交易量 | 盤前交易活躍度 | `premarket_volume` | ⭐⭐ |
| 買賣價差比例 | 流動性指標 | `(ask - bid) / mid_price` | ⭐⭐ |

**實際影響範例：**

```
股票 A（高流動性 - NVDA）：
- 日均交易量：300M 股
- 日均交易金額：$30B
- 財報利多發布 → 10分鐘內反應完畢
- 0.5小時最高價：準確預測

股票 B（低流動性 - 小型股）：
- 日均交易量：500K 股
- 日均交易金額：$10M
- 財報利多發布 → 需要 2-3 小時才反應完畢
- 0.5小時最高價：遠低於預測（因為還在反應中）
```

### 2. 市值相關特徵 ⭐⭐⭐

**為什麼重要：**
- 大型股：機構投資人多，反應較慢但穩定
- 小型股：散戶為主，反應快但波動大
- 微型股：流動性差，反應非常慢

**建議特徵：**

| 特徵名稱 | 說明 | 計算方式 | 重要性 |
|---------|------|---------|--------|
| 市值 | 公司總市值 | `shares * price` | ⭐⭐⭐ |
| 市值分類 | 大/中/小型股 | `log(market_cap)` | ⭐⭐⭐ |
| 流通股數 | 可交易股數 | `float_shares` | ⭐⭐ |
| 機構持股比例 | 機構投資人占比 | `institutional_ownership` | ⭐⭐ |
| 內部人持股 | 管理層持股 | `insider_ownership` | ⭐ |

**實際影響範例：**

```
大型股（市值 > $100B）：
- 例如：AAPL, MSFT, NVDA
- 機構持股：70-80%
- 反應速度：30-60分鐘（機構需要時間建倉）
- 反應穩定：波動較小，趨勢明確

中型股（市值 $2B-$100B）：
- 例如：SNOW, CRWD
- 機構持股：50-60%
- 反應速度：15-30分鐘
- 反應適中：波動中等

小型股（市值 < $2B）：
- 例如：IONQ, QBTS
- 機構持股：20-40%
- 反應速度：5-15分鐘（散戶快速反應）
- 反應劇烈：波動很大，可能過度反應
```

**為什麼重要：**

- 熱門題材 → 市場敏感度高

| 分析師覆蓋數 | 追蹤的分析師數量 | Financial Data | ⭐⭐ |
| 期權交易量 | 期權市場活躍度 | Options Volume | ⭐⭐ |
| 熱門股票排名 | 是否在熱門榜 | Trading Platform | ⭐⭐ |
| Google 搜尋趨勢 | 搜尋熱度 | Google Trends | ⭐ |

**實際影響範例：**

```

冷門股：

- 新聞報導：1 篇/天

- 財報發布 → 1-2小時才反應（需要消息擴散）
```

### 4. 時間相關特徵 ⭐⭐

**為什麼重要：**
- 開盤初期：交易最活躍，反應最快
- 盤中時段：交易減少，反應變慢
- 收盤前：交易再次活躍

**建議特徵：**

| 特徵名稱 | 說明 | 可能值 | 重要性 |
|---------|------|--------|--------|
| 財報發布時間 | 盤前 vs 盤後 | `before_market / after_market` | ⭐⭐⭐ |
| 距離開盤時間 | 消息發布到開盤的時間 | 小時數 | ⭐⭐ |
| 星期幾 | 週一到週五 | 1-5 | ⭐ |
| 是否月初/月底 | 機構調倉時間 | Boolean | ⭐ |
| 是否季度末 | 財報季 | Boolean | ⭐ |

**實際影響範例：**

```
盤前發布（7:00 AM）：
- 距離開盤：2.5 小時
- 市場有充分時間消化
- 開盤時反應已經大部分完成
- 0.5小時內：小幅調整

盤中發布（10:00 AM）：
- 正在交易時段
- 立即反應，波動劇烈
- 0.5小時內：主要反應完成

盤後發布（4:30 PM）：
- 距離下次開盤：16 小時
- 盤後交易量小，反應不完全
- 次日開盤：集中反應
- 0.5小時內：反應最劇烈
```

### 5. 波動性歷史特徵 ⭐⭐

**為什麼重要：**
- 高波動股：習慣快速反應
- 低波動股：反應較慢較穩定
- 歷史反應模式會重複

**建議特徵：**

| 特徵名稱 | 說明 | 計算方式 | 重要性 |
|---------|------|---------|--------|
| 歷史波動率 | 20日波動率 | `std(returns[-20:])` | ⭐⭐⭐ |
| 平均日內波動 | 日內高低價差 | `mean((high-low)/open)` | ⭐⭐ |
| Beta 值 | 相對大盤波動 | 回歸分析 | ⭐⭐ |
| 過去財報反應速度 | 歷史財報日的反應時間 | 統計過去 4 次財報 | ⭐⭐⭐ |
| ATR (Average True Range) | 真實波動範圍 | 技術指標 | ⭐⭐ |

**實際影響範例：**

```
高波動股（波動率 > 5%）：
- 例如：TSLA, NVDA, 小型成長股
- 日內波動：±10%
- 財報反應：快速且劇烈
- 0.5小時內：可能完成 80% 反應

低波動股（波動率 < 2%）：
- 例如：公用事業股、大型價值股
- 日內波動：±2%
- 財報反應：緩慢且穩定
- 0.5小時內：可能只完成 30% 反應

歷史反應模式：
- 過去 4 次財報平均反應時間：15 分鐘
- 本次預期：也會在 15 分鐘左右反應完畢
```

## 資料收集範例

### Python 程式碼範例

```python
import yfinance as yf
import pandas as pd

def get_liquidity_features(ticker, date):
    """
    獲取流動性相關特徵
    """
    stock = yf.Ticker(ticker)
    
    # 獲取歷史資料（過去 30 天）
    hist = stock.history(period='1mo', end=date)
    
    # 計算特徵
    features = {
        # 流動性特徵
        'avg_volume_20d': hist['Volume'][-20:].mean(),
        'avg_dollar_volume_20d': (hist['Volume'] * hist['Close'])[-20:].mean(),
        'relative_volume': hist['Volume'][-1] / hist['Volume'][-20:].mean(),
        
        # 市值特徵
        'market_cap': stock.info.get('marketCap', 0),
        'float_shares': stock.info.get('floatShares', 0),
        
        # 波動性特徵
        'volatility_20d': hist['Close'].pct_change()[-20:].std() * 100,
        'avg_daily_range': ((hist['High'] - hist['Low']) / hist['Open'])[-20:].mean() * 100,
        'beta': stock.info.get('beta', 1.0),
        
        # 其他特徵
        'institutional_ownership': stock.info.get('heldPercentInstitutions', 0) * 100,
    }
    
    return features

# 使用範例
ticker = 'NVDA'
date = '2026-02-26'
features = get_liquidity_features(ticker, date)
print(features)
```
